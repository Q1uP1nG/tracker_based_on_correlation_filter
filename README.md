# Simple Correlation Filter Tracker

基于 Python + OpenCV 实现的简易目标跟踪程序，适合作为课程实验代码、算法练习代码。

包含两个核心文件：

- `run.py`：程序入口，负责读取视频/摄像头、手动框选初始目标、调用跟踪器并显示结果
- `basetracker.py`：跟踪器主体，负责特征提取、相关滤波响应计算、尺度候选搜索、目标位置更新与模板更新

---

## 1. 项目功能

该程序实现了一个基础的相关滤波目标跟踪流程：

1. 用户在第一帧手动框出目标
2. 跟踪器根据初始框建立目标模板
3. 后续每一帧中，在目标附近区域搜索最可能的位置
4. 输出新的目标框并实时显示 FPS

程序支持：

- 从摄像头读取画面
- 从视频文件读取画面
- 手动初始化目标框
- 实时显示目标跟踪结果
- 实时显示运行速度（FPS）

---

## 2. 文件说明

## `run.py`

`run.py` 的职责非常直接：它不负责“怎么跟踪”，只负责“怎么运行整个流程”。

它主要完成以下几件事：

### (1) 读取输入

程序启动时会根据命令行参数决定输入来源：

- 不带参数：默认打开摄像头
- 带一个数字参数：打开对应编号的摄像头
- 带一个字符串参数：将其视为视频路径并读取视频文件

也就是说，这个文件主要解决的是“数据从哪里来”的问题。

### (2) 创建跟踪器

程序中实例化了：

```python
tracker = basetracker.BaseTracker(False, True, False)
```

这里真正执行目标跟踪的是 `BaseTracker`，`run.py` 只是把它创建出来并调用。

### (3) 手动选择初始目标框

在第一帧中，程序使用 OpenCV 的 `cv2.selectROI()` 让用户手动框出目标区域。

这一步非常关键，因为这个项目不是自动检测目标，而是“给定初始框后进行持续跟踪”。

### (4) 逐帧更新目标框

初始化完成之后，程序会在每一帧调用：

```python
boundingbox = tracker.update(frame)
```

然后把更新后的框画到图像上。

### (5) 显示 FPS

程序会统计每一帧的处理时间，并用滑动平均的方式估计当前 FPS，从而更稳定地显示实时速度。

---

## `basetracker.py`

`basetracker.py` 是整个项目的核心，它决定了“目标到底怎么跟踪”。

如果说 `run.py` 管的是流程，那么 `basetracker.py` 管的就是算法本体。

它实现的是一个基于相关滤波思想的简化跟踪器，并在基础版本上做了一些工程化增强。

---

## 3. 跟踪器整体思路

这个跟踪器的核心思路可以概括为：

**在当前目标附近截取搜索区域 -> 提取特征 -> 和模板做相关匹配 -> 找到响应最高的位置 -> 更新框的位置与大小 -> 用新结果缓慢更新模板。**

---

## 4. `BaseTracker` 的主要组成部分

## 4.1 初始化参数

在 `__init__()` 中定义了很多跟踪相关参数，例如：

- `self._padding`：搜索区域相对目标框的扩展比例
- `self._interp_factor`：模板更新速度
- `self._sigma_factor`：高斯核相关中的参数
- `self._lambda`：频域求解时的正则项
- `self._tmpl_sz`：模板统一缩放后的尺寸
- `self._scale_pool`：常规尺度候选集合
- `self._rescue_scale_pool`：低置信度时使用的更大尺度搜索集合

---

## 4.2 ROI 与图像区域处理

文件里有一批辅助函数专门用于处理矩形框和图像裁剪，例如：

- `limit()`
- `getBorder()`
- `subwindow()`

它们的作用是：

- 保证框不要越界
- 如果框超出图像边界，就自动补边
- 正确截取搜索区域

这部分虽然不是算法核心，但非常重要。因为目标在运动时经常会靠近图像边缘，如果没有这些保护，程序很容易崩掉或者裁剪出错。

---

## 4.3 特征提取

跟踪器没有直接拿原始彩色图像去匹配，而是先做了一个比较轻量的特征提取：

1. 如果输入是彩色图像，先转灰度
2. 将灰度值归一化
3. 用 Sobel 算子计算水平和垂直梯度
4. 计算梯度幅值
5. 将灰度信息与梯度信息按权重混合

对应思想是：

- 灰度信息保留目标的大致外观
- 梯度信息强调边缘和结构

最终得到的特征图比直接用原图稍微更稳一些，尤其在目标边缘较明显时更有帮助。

---

## 4.4 频域相关计算

这是整个程序最关键的理论部分之一。

### 为什么要用相关滤波

如果在每一帧中都用“滑窗 + 暴力比较”的方式在周围搜索目标，那么计算量会很大。

相关滤波的核心思想是：  
把原本在空间域里比较复杂的相关运算，转移到频域中去做，从而提升计算效率。

代码中相关的函数包括：

- `fftd()`：做傅里叶变换
- `complexMultiplication()`：复数乘法
- `complexDivision()`：复数除法
- `gaussianCorrelation()`：计算高斯核相关响应

这里体现的是“相关滤波 + 频域加速”的经典思路。

---

## 4.5 高斯响应与滤波器学习

初始化时，程序会生成一个高斯峰：

```python
self._prob = self._create_gaussian_peak(...)
```

它表示一种理想状态：  
如果目标对齐得很好，那么响应图应该在目标中心附近出现一个明显峰值。

之后在 `update_model()` 中，程序用当前样本去更新滤波器参数。  
本质上就是让模型逐渐学会：什么样的区域像目标，什么样的区域不像目标。

因为使用了插值更新，所以模板不会被新一帧完全替换，而是缓慢适应目标外观变化。

这有利于：

- 跟随外观缓慢变化
- 避免偶然噪声把模板带偏

---

## 4.6 多尺度候选搜索

这部分是该实现里比较明显的改良点之一。

程序不是只在当前位置搜“平移”，而是会同时尝试多个尺度候选。例如：

```python
self._scale_pool = [0.93, 0.97, 1.00, 1.03, 1.05, 1.08, 1.12]
```

也就是说，它会把搜索区域按多个比例缩放后分别计算响应，谁的得分最好就优先考虑谁。

这样做的意义是：

- 当目标轻微变大或变小时，跟踪器更容易跟上
- 不至于一直死守固定框大小

同时，程序还设计了一个 `score`，把以下信息综合起来：

- 响应峰值 `peak_value`
- PSR（峰值旁瓣比）
- 尺度惩罚项

这样就不是单纯看“峰值最高”，而是综合考虑“匹配是否明显、尺度变化是否合理”。

---

## 4.7 PSR：判断这次匹配靠不靠谱

PSR（Peak-to-Sidelobe Ratio）可以理解为：

**主峰到底是不是明显高于周围噪声。**

如果响应图中最高点虽然高，但周围也都差不多高，那说明这次匹配并不可靠。  
如果最高点远高于周围其他位置，说明目标定位更可信。

在这个项目中，PSR 被用于：

- 判断当前跟踪结果是否可信
- 决定是否进入低置信度恢复状态
- 决定是否更积极地更新模板

这比只看最大响应值更稳一些。

---

## 4.8 运动预测与搜索区域扩大

程序维护了一个简单速度量：

```python
self._velocity
```

它用于估计目标下一帧大概会往哪里移动。

在更新时，程序不是死盯当前中心，而是先用速度预测一个 `predicted_center`，然后围绕这个预测位置建立搜索区域。

当以下情况出现时，程序还会扩大搜索范围：

- 处于前几帧 warmup 阶段
- 目标速度较快
- 连续低置信度跟踪

这是一种很常见的恢复策略：  
**当不太确定目标在哪时，就扩大搜索范围再找。**

---

## 4.9 尺度更新与“抗缩小”倾向

在 `update()` 中，程序会根据当前最优尺度候选更新框大小。  
不过它不是直接一步跳到新尺度，而是做了平滑处理：

```python
blended_ratio = 1.0 + (scale_factor - 1.0) * self._scale_smooth
```

这意味着框大小变化会比较缓慢，不容易突然抖动。

此外，代码中还保留了一种明显的设计倾向：  
**对缩小会更谨慎。**

原因很实际：

- 一旦框缩得太小，目标周围一些有辨识度的边缘信息可能被裁掉
- 跟踪器容易逐渐退化成只盯住局部高响应区域
- 最后可能出现“只跟到目标一小块”的问题

所以这个实现里对“越跟越小”是比较防备的。

---

## 4.10 模板更新

当这次跟踪结果可信时，程序会再次从新位置提取特征，并更新模板：

```python
self.update_model(x, lr)
```

这里的学习率 `lr` 也会根据当前置信度做调整：

- 普通可信：正常更新
- 非常可信：略微更积极更新
- 不可信：不更新或弱更新

这样做的目的是尽量减少“模板污染”。

所谓模板污染，就是把错误位置当成真目标去更新模板，结果模型越学越偏，最后彻底丢失目标。

---

## 5. 程序运行流程总结

整个项目的实际流程可以总结为：

1. 读取视频帧
2. 用户手动框选初始目标
3. 用初始目标建立模板
4. 每一帧中：
   - 根据上一帧位置和速度预测搜索中心
   - 构造搜索区域
   - 在多个尺度下计算响应
   - 选出最优位置和尺度
   - 更新目标框
   - 如果结果可信，则更新模板
5. 将目标框和 FPS 画到画面上并显示

## 6. 如何运行

### 环境依赖

建议使用 Python 3，并安装：

```bash
pip install opencv-python numpy
```

### 运行方式

#### 方式 1：摄像头输入

```bash
python run.py
```

#### 方式 2：指定摄像头编号

```bash
python run.py 0
```

或

```bash
python run.py 1
```

#### 方式 3：读取视频文件

```bash
python run.py your_video.mp4
```

5. 按 `q` 或 `Esc` 退出
